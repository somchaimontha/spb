### 3.2 Sheet: ประมาณการรายจ่าย

**โครงสร้าง Header (3 แถวแรก):**
```
แถว 1: ชื่อรายงาน + ปีงบประมาณ + ชื่อสถานศึกษา
แถว 2: กลุ่มคอลัมน์งบประมาณ (เงินงบประมาณ | เงินอุดหนุน | งบรายจ่ายอื่น | เงินบกศ.)
แถว 3: ชื่อคอลัมน์ย่อย (ปวช. | ปวส. | ระยะสั้น | อุดหนุน | หนังสือ | ฯลฯ) + รวม

```
รับมา      →  ยอดงบที่ได้รับจัดสรรทั้งหมดแต่ละประเภท

```

```
รวม        = SUM(ไตรมาส 1 + 2 + 3 + 4)

### 3.5 Sheet: จัดสรร งปม. (Budget Distribution)

**โครงสร้าง:**
```
Header:
  แถวสรุปบน:
    รับมา      = ดึงจาก Sheet งปม.เข้า (ยอดรวมรายรับ)
    จัดสรรไป   = SUM ของทุกรายการในชีตนี้
    คงเหลือ    = รับมา − จัดสรรไป

รายการ (แบ่งตามหมวด 1xx–9xx เหมือน Sheet ประมาณการ):
  ปีงบ | รหัส | ชื่อรายการ | [คอลัมน์งบ] | รวม | ใช้ไปแล้ว | ร้อยละ | เหลือ
```

**คอลัมน์สำคัญต่อรายการ:**
| คอลัมน์ | สูตรคำนวณ | คำอธิบาย |
|---------|----------|----------|
| รวม | SUM(คอลัมน์งบทุกประเภท) | งบที่จัดสรรให้รายการนี้ |
| ใช้ไปแล้ว | อ้างอิงจากรายการเบิกจ่ายจริง | ยอดใช้จ่ายจริง |
| ร้อยละ | ใช้ไปแล้ว / รวม × 100 | % การใช้งบ |
| เหลือ | รวม − ใช้ไปแล้ว | ยอดคงเหลือ |

**ยอดสรุประดับบน:**
```
รับมาทั้งสิ้น  = 17,462,832.23 บาท
จัดสรรไป      = 14,236,526.38 บาท
คงเหลือ       =  3,226,305.85 บาท
ใช้ไปแล้ว     =  8,548,554.43 บาท  (60.05%)
```

## 8. ข้อสังเกตสำหรับการออกแบบ Web App

### 8.0 หลักการแปลงโครงสร้าง Excel → Web App

| Excel (เดิม) | Web App (ใหม่) | เหตุผล |
|-------------|--------------|--------|
| Sheet แยกต่างหากต่อรหัส (101, 202...) | ตาราง `disbursements` ตารางเดียว กรองด้วย `item_code` | จัดการง่าย ค้นหาได้ ไม่ต้องสร้าง Sheet ใหม่ทุกครั้ง |
| คลิก Tab Sheet เพื่อดูรายการ | คลิก Icon รหัส → Pop-up/Drawer แสดงรายการ | UX ดีกว่า ใช้งานบนมือถือได้ |
| ยอดสรุปบน Sheet คำนวณใน Excel | คำนวณ real-time จาก `disbursements` | ข้อมูล live ไม่ต้องอัปเดตสูตร |
| Sheet ว่าง (102, 201...) | แถวใน `budget_distribution` ที่ `actual_used = 0` | ไม่เปลืองพื้นที่ แต่ยังอ้างอิงได้ |
| ปุ่ม "กลับไปหน้าจัดสรร งปม." | Breadcrumb navigation: จัดสรร งปม. > รหัส 101 | UX มาตรฐาน |
| รหัสลำดับ 001–099 | Auto-increment ต่อ `item_code` | ป้องกันซ้ำ |

1. **ยอดใช้จริงในโครงการ** — ในไฟล์ Excel บางโครงการมีตัวเลขยอดใช้จริงบันทึกไว้ในคอลัมน์ขวาสุดของชื่อผู้รับผิดชอบ (เช่น `103610`, `156000`, `498000`) ซึ่งไม่มีหัวคอลัมน์ชัดเจน → ควรเพิ่ม field `actual_used` ในระบบใหม่

2. **โครงการบางรายการมี `***`** — หมายถึงยังไม่กำหนดงบประมาณ หรืองบขึ้นอยู่กับการจัดสรรจริง → ควรมีสถานะ `TBD` หรือ `รอจัดสรร`

3. **รหัสโครงการ** — ปัจจุบันใช้ลำดับที่ (1–121) ต่อเนื่องทั้งสถานศึกษา ไม่แยกตามฝ่าย → ควรออกแบบรหัสใหม่เป็น `[ปีงบ]-[รหัสฝ่าย]-[ลำดับ]` เช่น `2569-BUD-001`

4. **ไตรมาสในรายการจัดสรร** — งบจำนวนมากได้รับในไตรมาสที่ 1 (ต.ค.) ทำให้ไตรมาส 2–4 ว่างเปล่า → Web App ควรรองรับการบันทึกแบบไม่ครบไตรมาส

5. **ยอด `#DIV/0!`** — เกิดจากหารด้วยศูนย์ในรายการที่ยังไม่มีการจัดสรร → Web App ต้องจัดการ edge case นี้ด้วย

6. **รายการที่มีค่าลบ** — เช่น ยอดคงเหลือของ `จัดการเรียน = -77,889` แสดงว่าจัดสรรเกินงบที่รับมา → ต้องมีการแจ้งเตือน Overspend

### Google Sheets Database Schema (สรุปทุก Table)

```
SHEET: settings          ← Note + ข้อมูลปีงบ + ชื่อสถานศึกษา + ค่าตั้งค่าทั้งหมด
SHEET: users             ← ผู้ใช้, รหัสผ่าน (hash), role, สิทธิ์
SHEET: divisions         ← ฝ่ายงาน, งาน, แผนกวิชา (จาก Excel ฝ่ายงาน 4 ฝ่าย)
SHEET: budget_categories ← หมวดการเงิน 1xx–9xx (dynamic จาก Settings)
SHEET: budget_types      ← ประเภทเงิน ปวช./ปวส./บกศ./ฯลฯ (dynamic)
SHEET: project_statuses  ← สถานะโครงการ + สี (dynamic)

SHEET: budget_estimates   ← ประมาณการรายจ่าย (จาก Sheet ประมาณการรายจ่าย)
SHEET: budget_allocations ← รายการจัดสรรรายไตรมาส (จาก Sheet รายการจัดสรร)
SHEET: budget_received    ← รายการรับงบเข้า + รหัสเอกสาร (จาก Sheet งปม.เข้า)
SHEET: budget_distribution← ยอดจัดสรรแต่ละหมวด + ใช้ไป + เหลือ (จาก Sheet จัดสรร งปม.)

SHEET: projects          ← ทะเบียนโครงการ (จาก Sheet โครงการในแผน 2569)
SHEET: project_history   ← ประวัติการแก้ไขชื่อโครงการ + งบ (ใหม่)
SHEET: project_cancellations ← การยกเลิก/ยุบรวมโครงการ (ใหม่)
SHEET: execution_rounds  ← การดำเนินโครงการแต่ละครั้ง + running balance (ใหม่)
